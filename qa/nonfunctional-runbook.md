# Non-Functional Runbook

## 1. 性能試験（SLO）
- 一覧検索（世帯/取引/行事）: p95 <= 2.0s
- 詳細表示: p95 <= 2.5s
- CSV/帳票生成: p95 <= 5.0s

### 手順
1. テストデータを1,000件規模に調整
2. 同一クエリを30回実行
3. p50/p95/p99 を記録
4. SLO未達ならクエリ/インデックス確認

## 2. 信頼性
### 手順
1. 書込中にDB接続障害を擬似発生
2. 部分保存がないことを確認
3. バックアップからリストア演習（RTO/RPO記録）

## 3. セキュリティ
### チェック
- 認可: 他IDアクセス（IDOR）
- 入力: XSS文字列反映
- セッション: ログアウト後トークン無効
- CSRF: 変更系の保護方針確認

## 4. プライバシー
- ログに個人情報（住所/電話/金額詳細）が過剰出力されない
- CSV出力のロール制御確認

## 5. データ品質
- 取引は event/deceased/household いずれか必須
- お布施/塔婆料の集計一致
- 重複レコード率を計測
